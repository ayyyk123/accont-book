# 가계부 36.9차 — 모바일 글자·버튼 배열 전면 정리

## 카드 대신 결제 예산 처리

카드 대신 결제로 받을 돈을 등록한 경우:

- 카드 사용액과 카드 결제예정액은 실제 카드 결제액 전체로 유지
- 내 예산 또는 목적자금에서는 받을 금액만큼 즉시 제외
- 실제로 입금받기 전이어도 받을 돈으로 약속한 금액은 예산을 사용하지 않은 것으로 계산
- 일부 수령 후에도 이미 받은 금액과 남은 받을 금액은 계속 예산에서 제외
- `받지 않기로 한 금액`은 다시 내 예산 사용액에 포함

예: 카드 결제 100,000원, 받을 돈 40,000원

- 카드값: 100,000원
- 예산 사용액: 60,000원
- 받을 돈: 40,000원

## 기간 바로가기 버튼 삭제

- 일반대출 이자 상환내역의 `올해` 버튼 삭제
- 통장별 전체 거래내역의 `이번 달` 버튼 삭제
- 시작일과 종료일은 기존처럼 직접 선택

## 통장목록 버튼과 모바일 배열

- 각 통장 카드에 잘못 추가된 일반 `이체 예정` 버튼 삭제
- 입금·출금·지출·소득·이체·거래내역·수정·삭제 유지
- 통장 메타정보를 버튼과 별도 줄에 배치
- 은행명과 설명이 한 글자씩 세로로 내려가던 모바일 배열 수정

## 자동 잔액 보충 안내

통장 잔액 예상에서 카드 결제통장에만 제한되던 보충 안내를 모든 통장에 적용했습니다.

- 카카오 마이너스통장 포함
- 예정된 카드값·고정지출·기타 출금 후 잔액이 0원 아래로 내려가는 날짜 감지
- 다른 마이너스통장의 사용 가능 한도가 있으면 기존 `다른 마이너스통장에서 이체 예정` 안내 표시
- 통장 카드에 별도 버튼을 넣지 않고 잔액 예상화면에서 필요한 때만 표시

## 수기 이체 예정 등록

`통장,카드관리 > 통장목록` 상단에 `+ 이체 예정 등록` 버튼을 추가했습니다.

입력 항목:

- 보내는 통장
- 받는 통장
- 이체 예정일
- 이체 금액
- 메모

저장한 이체 예정은:

- 완료 전 실제 잔액에는 반영되지 않음
- 각 통장의 잔액 예상과 미반영 예정항목에 표시
- 예정일 당일 또는 날짜가 지나면 최상단 이체 알림에 표시
- `이체 완료` 처리 시 두 통장 잔액에 반영
- 완료한 날에는 `오늘 완료한 이체`에 계속 표시

## 이체 알림 없음

오늘 미완료 이체와 오늘 완료한 이체가 모두 없어도 최상단 알림 영역을 숨기지 않습니다.

- 제목: `이체 알림 없음`
- 안내: `오늘 예정되거나 오늘 완료한 이체가 없습니다.`
- 건수: `0건`

## 유지 기능

- 카드 대신 결제와 받을 돈 자동 연동
- 미완료·오늘 완료 이체 알림 및 접기·펴기
- 대출별 내역과 통장 전체 거래장부
- 투자 평가 기준일 보존
- 한글 입력 수정
- 월·일 선택 조회
- All 고정필터
- Firebase 데이터와 백업 구조
