# 가계부 35.4차 — 통장 입출금·투자거래 연결

## 잘못된 처리 방식 수정

투자원금 출금은 소비가 아니므로 `통장 지출`로 기록하지 않습니다.
35.4차부터 투자·현금 이동은 별도의 `통장 입금 / 통장 출금`으로 기록하며
월 소득·월 소비·생활비 예산에서 모두 제외합니다.

## 통장 입금·출금

- All 탭에 `+ 통장 출금` 추가
- 통장 입금: 통장 잔액 증가, 월 소득에서는 제외
- 통장 출금: 통장 잔액 감소, 월 소비·예산에서는 제외
- 입금·출금 모두 거래 상대를 `현금계좌` 또는 `투자`로 선택
- 통장 종류에 `현금계좌` 추가
- 현금계좌 연결 시 양쪽 계좌 잔액을 동시에 반영
  - 통장 출금 → 현금계좌 입금
  - 현금계좌 출금 → 통장 입금
- All 필터와 달력에서 통장 입금·출금을 각각 조회
- 통장 장부와 미래 예상잔액에도 입출금 반영

## 투자거래 연결

- 통장 입금·출금에서 투자 종류 선택
  - 경매투자
  - 주식투자
  - 기타투자
- 선택한 종류에 등록된 투자 거래처만 표시
- 투자 출금은 해당 투자처의 총 투자원금에 자동 합산
- 투자 입금은 해당 투자처의 총 회수금에 자동 합산
- 투자처를 삭제할 때 연결된 입출금이 있으면 삭제 방지

## 투자 입금 금액 계산

투자 거래처에서 통장으로 입금받을 때 아래 금액을 따로 입력합니다.

- 회수 원금
- 수익
- 손실

실제 입금액은 자동으로 다음과 같이 계산됩니다.

`회수 원금 + 수익 - 손실 = 실제 통장 입금액`

예시:
회수 원금 1,000만원 + 수익 200만원 - 손실 50만원 = 실제 입금 1,150만원

## 기존 투자기록

- 투자 등록 화면의 투자원금·회수금은 앱 사용 전 기존 금액을 입력하는 항목
- 이후 연결한 통장 출금·입금은 자동 합산
- 총 손익 = 현재 평가금액 + 총 회수금 - 총 투자원금
- 연결된 수익·손실 합계와 거래 건수 표시

## 호환성

- 기존 35.3차 투자·통장입금 기록 자동 호환
- 기존 목적자금 분류필터와 카드값 기간조회 유지
- 화면·백업·PWA 캐시를 모두 35.4차로 통일
