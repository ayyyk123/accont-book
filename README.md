# 가계부 36.3차 — 현금계좌 이체 한글 입력 수정

## 수정 내용

현금계좌를 만든 뒤 통장이체를 입력할 때 `항목`과 `메모`에서
한글 입력이 영문 자판처럼 처리되는 문제를 수정했습니다.

- 항목·메모를 한국어 텍스트 입력으로 명시
- 한국어 지원 글꼴 우선 적용
- 한글 조합 시작·진행·완료 이벤트 처리
- 한글 조합 중 자동 항목입력이 값을 덮어쓰지 않도록 차단
- 사용자가 항목을 직접 입력하면 이후 현금계좌명 자동입력이 덮어쓰지 않음
- 통장거래 폼이 다시 생성되어도 한글 입력 설정을 자동 재설치

## 실제 검사

Chromium에서 한글 조합 이벤트를 발생시켜 다음을 확인했습니다.

- 항목 `한글 이체` 유지
- 메모 `현금계좌 메모` 유지
- 입력언어 `ko`
- 입력모드 `text`
- 한글 입력 중 자동 항목 덮어쓰기 없음

## 유지 기능

- 36.2차 통장목록 이체 버튼
- 36.1차 월·일 선택 조회
- 36차 All 고정필터
- 투자 독립 탭
- Firebase 데이터와 백업 구조
